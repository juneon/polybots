TL;DR — 2026-07-14 심야 스크리닝에서 MA 기울기 필터(entry_slope_max)가 간이 리플레이 기준 +2.96 → +43.40으로 세 소스 전부 개선되는 최유망 신규 변수로 나왔다. 전략 구현은 완료 상태(기본 null = 현행 불변)이므로, 이번 작업은 D23 채택 절차의 필수 단계인 실엔진 그리드 검증이다: 기존 6축에 slope 축 + 예약분(cap 0.6~0.7, ma_len 600)을 더한 24,192조합 풀 그리드. config 반영은 D23 3중 기준(train 상위 · val≥0 · 이웃 plateau) 충족 시에만.

1. 배경

  • P1 판정(2026-07-07): MA는 라이브 부적합 (mar03 대패). 단 7월 sim 레짐에서는 MA가 threshold보다 우세 (07-14 실매매 리뷰: MA +19.06 vs threshold −5.10) — 레짐 뒤집힘.
  • 07-14 심야 스크리닝: 크로스업 진입 669건 중 상승 MA 추격 진입(slope>0, 120건)이 −34.8로 주된 독소. "하락 중인 MA를 뚫는 반등"(slope ≤ −0.005)만 진입 시 janfeb +23.8→+51.0 / mar03 −21.8→−12.3 / sim +1.0→+4.7.
  • 단 스크리닝은 간이 리플레이(양 사이드 독립 포지션 등 실전략과 다름) — 채택 전 실엔진 스윕이 D23 필수 절차.
  • 07-14 회사 그리드의 잔여 과제: 어제 후보 cap0.7/ma600이 그리드 축 밖이었음 → 이번에 축 포함.

3. 판정 기준 (D23 — 사전 고정)

  1. train(janfeb) 점수 상위
  2. val(mar03+sim) ≥ 0
  3. 이웃 plateau — 특히 ma_len 600 절벽 여부(07-14 그리드에서 240 절벽 −14.9 전례)와 slope 축 단조성
셋 다 충족하는 조합이 있어도 MA sim 재가동은 별개 결정(recorder 분리 P3와 연계) — 이번 산출물은 "차기 재판정(완전 sim 60)의 비교 대상 등록"까지. config 즉시 교체는 하지 않는 것이 기본값 (07-14 결론: 34 slugs 표본으로 교체 보류 — 현재도 sim 완전 28로 동일 상황).

4. 가설 (사전 등록)

  • H1: slope ≤ 0 계열이 none 대비 train·val 동시 개선 (스크리닝 재현). 실전략(단일 포지션·재진입 규칙 포함)에서 크기는 줄어들 수 있음.
  • H2: cap 0.6~0.7 / ma_len 600 영역(07-13 심야 전 구간 생존형 후보)은 slope 필터와 중복 효과라 결합 시 한쪽이 무의미해질 수 있음.
  • H3: 07-14 후보 영역(cap0.45/ma200/tp0.99)은 slope 축 추가 후에도 생존하면 신뢰도 상승.
