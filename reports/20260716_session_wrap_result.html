4. P2 — live SELL 스윕 스레드 분리

문제(07-15 분석): exit_tp no-match 시 IOC 스윕이 고정가로 창 전체(기본 10초)를 소진하며 메인 루프를 블로킹 — 시세·heartbeat·판단이 전부 멈춤.

설계 요점내용
즉시 반환SELL 시 fill()이 스윕을 워커 스레드로 시작하고 곧바로 submitted(reason sell_sweep_async) 반환. execution.sell_sweep_async 기본 on (끄면 기존 동기 동작)
재진입 방지스윕 진행 중 새 SELL은 rejected(reason sell_inflight) — 양 전략 모두 filled 아니면 exit 래치 유지라 스윕 종료 후 자연 재시도
SOT 단일화워커는 account를 만지지 않음. 완료 trade는 runner의 drain_async_trades가 매 tick·slug 경계·종료 시 회수해 account.apply → on_trade → logger 순서로 반영. sync_position도 회수 시점(메인 스레드)에 수행
종료 flushfinally에서 스윕 join(창+2초) 후 잔여 결과 반영 — 정지 중 체결 유실 방지
불변식SPEC "Executor 무상태" → "트레이딩 상태 없음(스레드 핸들·결과 큐는 운영 상태 예외)"로 정정
검증테스트 3개 추가(submitted→poll 전달 · inflight 거부/재시도 · drain 반영) — 전체 83개 통과. 라이브 실측 확인은 다른 P2 항목과 함께 잔여
